# 创建 Obsidian 笔记

你需要帮助用户在 Obsidian vault 中创建一个新笔记。请按照以下步骤操作：

## 参数
- 笔记标题：$ARGUMENTS 的第一个参数（如果提供）
- 模板类型：$ARGUMENTS 的第二个参数（可选：daily、meeting、tech、reading）

## 步骤

1. **确定笔记信息**：
   - 如果用户没有提供标题，询问笔记标题
   - 询问是否需要使用模板（日记、会议笔记、技术笔记、读书笔记）
   - 询问笔记应该放在哪个目录下

2. **生成 frontmatter**：
   ```yaml
   ---
   title: 笔记标题
   created: YYYY-MM-DD HH:mm:ss
   updated: YYYY-MM-DD HH:mm:ss
   tags: []
   aliases: []
   ---
   ```

3. **应用模板**：
   - **日记模板**：
     ```markdown
     # YYYY-MM-DD

     ## 今日目标
     -

     ## 今日笔记


     ## 今日总结


     ## 明日计划
     -
     ```

   - **会议笔记模板**：
     ```markdown
     # 会议：[标题]

     **时间**：YYYY-MM-DD HH:mm
     **参与者**：
     **议题**：

     ## 讨论内容


     ## 决策事项
     -

     ## 行动项
     - [ ]

     ## 后续跟进

     ```

   - **技术笔记模板**：
     ```markdown
     # [技术主题]

     ## 概述


     ## 核心概念


     ## 示例代码

     ```language

     ```

     ## 参考资料
     -

     ## 相关笔记

     ```

   - **读书笔记模板**：
     ```markdown
     # [书名]

     **作者**：
     **出版年份**：
     **阅读日期**：YYYY-MM-DD

     ## 核心观点


     ## 重要摘录


     ## 个人思考


     ## 行动计划
     -
     ```

4. **智能标签建议**：
   - 根据笔记标题和内容分析可能的标签
   - 建议 3-5 个相关标签
   - 询问用户是否添加这些标签

5. **创建笔记文件**：
   - 使用 Write 工具创建笔记文件
   - 文件路径格式：`vault路径/目录/笔记标题.md`
   - 确保文件名符合 Obsidian 规范（避免特殊字符）

6. **确认创建**：
   - 告诉用户笔记已创建
   - 提供笔记的完整路径
   - 建议相关的操作（如添加链接、创建相关笔记等）

## 注意事项
- 文件名应避免使用特殊字符：`/ \ : * ? " < > |`
- 确保目标目录存在，如不存在则创建
- frontmatter 中的时间使用 ISO 格式
- 提供有帮助的标签建议，但不要过多

## 示例用法
```
/note-create "学习 React Hooks" tech
/note-create "2025-01-15 日记" daily
/note-create "产品讨论会议"
```
